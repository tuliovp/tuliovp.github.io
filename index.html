This Very Website (2021)

An HTML project to store all my projects

GitHub

Description

This is an entry-level project to build an environment where I can expose my projects' portfolio, both in Web Development and Data Science, besides practicing the following skills:

  • HTML
  • CSS
  • JavaScript
  • Design / UX

HTML

I have followed the best practices I found in repositories on Github, in terms of code style and explanatory comments. Highlights from the html file include:

  • Meta tag that gives the browser instructions on how to control the page's dimensions and scaling;
  • Meta tag used to summarize the web page's content - Search engines use it in search results to let visitors know what a page is about before they click on it;
  • Google Tag Manager, to identify visitor behavior within the site;
  • Visitors counter, from a third-party provider;
  • Spotify embedded playlist;

CSS

Despite its simplicity, there is a fair amount of code running behind it (~1500 lines of css), including:

  • All dimensions to fit different screen sizes;
  • WebKit extensions to CSS & CSS [attribute] Selector;
  • Font APIs;
  • Toggles with adjustments;
  • Progress bar animations;
  • Two different types of modals/lightboxes;
  • Gallery of divs;
  • Hover overlay effects;
  • Filtering buttons;

JavaScript

I have chosen to work with pure js, when jQuery was the mainstream go-to option. Pure Javascript to access the DOM can be faster as you can cut the overhead that jQuery has on this. You can find in the js file functions working on the following actions:

  • Show email when clicked;
  • Scroll down slowly when clicked in Notes / Cases;
  • Modal to work with different files in the same webpage;
  • Next/previous controls;
  • Thumbnail image controls;
  • Show/hide filtered elements;
  • Highlight active class to the current control button;
  • Close modal when the user clicks anywhere outside it or clicks on (x);
  • Song player

Design / UX

I thought of building a fairly simple site, approaching a level of minimalism that I fancy on other webpages out there. The main challenge was to bootstrap it all from scratch, without using any template as a base (although having several inspirations, obviously).

A few design goals of mine:

  • Monochromatic, give the impression of a paper CV;
  • Personal touch (Notes / Cool stuff / Jam / Cinema) - not just the cheesy professional stuff;
  • Divs instead of "a" to increase the click area - and sometimes vice-versa;
  • Cursor style changes depending on what is clickable or not (it might get confusing sometimes);

Playing Around (React) 2021

Giving the JavaScript library a try

App

Description

To create apps with interactive UIs that will render data updates, React can be really helpful.

First I load and unpack the version of react and react-dom, keeping the crossorigin attribute set. Then, I add an attribute to script tag for babel to translate this code to AJAX and create a Class component called App. Components let you split the UI into independent, reusable pieces, and think about each piece in isolation. They accept arbitrary inputs (called “props”) and return React elements describing what should appear on the screen.

Components serve the same purpose as JavaScript functions, but work in isolation and return HTML via a render() function. In this case, it will render simple random addition problems and the "You won!" message if(this.state.score === 5).

I have created different events which will be updated when the Enter key is inputKeyPress - extracting the answer and checking if it is correct. If so, score: state.score + 1. If not, score: state.score - 1.

Playing Around (Django) 2021

Trying Python’s Django framework to create dynamic applications that generates HTML and CSS.

Airline App

Description

The goal is to practice not only the framework but databases and query skills using SQL. For this, I've created an Airline app which we can add new airports and flights information, and users can log in/out to check their booking details - all saved into the main Airline database.

Basic files and directories:

Project:

  • manage.py is what we use to execute commands on our terminal;
  • settings.py contains some important configuration settings for our new project;
  • urls.pycontains directions for where users should be routed after navigating to a certain URL.
App:
  • views.py.contains a number of different views, and we can think of a view as one page the user might like to see. To create the view, I write a function that takes in a request rendering layouts or returning HttpResponses;
  • templates directory allow us to write HTML and CSS in separate files and render those files using Django. In this context, we can provide information that we would like to have available within our HTML files. Django’s templating language gives us a way to eliminate poor design: the template inheritance. This allows us to create and extend a layout.html file that will contain the general structure of our page and the {% block body %} we want to render.

Styling

If we want to add a CSS static file, we’ll first create a folder called static, then create the airline apps folders within that, and then a styles.css file within that. Now, to include this styling in our HTML file, we add the line {% load static %} to the top of our HTML template, which signals to Django that we wish to have access to the files in our static folder.

Project and apps

Django Models are a level of abstraction on top of SQL that allow us to work with databases using Python classes and objects rather than direct SQL queries.
Now, rather than creating actual paths and getting started on views.py, we’ll create some models in the models.py file. In this file, we’ll outline what data we want to store in our application and create a new model that extends Django’s model class. Then, Django will determine the SQL syntax necessary to store information on each of our models.

class Airport(models.Model):
   code = models.CharField(max_length=3)
   city = models.CharField(max_length=64)

   def __str__(self):
      return f"{self.city} ({self.code})"

class Flight(models.Model):
   origin = models.ForeignKey(Airport, on_delete=models.CASCADE, related_name="departures")
   destination = models.ForeignKey(Airport, on_delete=models.CASCADE, related_name="arrivals")
   duration = models.IntegerField()

   def __str__(self):
      return f"{self.id}: {self.origin} to {self.destination}"

To create a database from our models, we navigate to the main directory of our project and run the command.

Booking

Now, let’s work on giving visitors to our site the ability to book a flight. We’ll do this by adding a booking route in urls.py:

path("<int:flight_id>/book", views.book, name="book")

Now, we’ll add a book function to views.py that adds a passenger to a flight:

def book(request, flight_id):

   # For a post request, add a new flight
   if request.method == "POST":

      # Accessing the flight
      flight = Flight.objects.get(pk=flight_id)

      # Finding the passenger id from the submitted form data
      passenger_id = int(request.POST["passenger"])

      # Finding the passenger based on the id
      passenger = Passenger.objects.get(pk=passenger_id)

      # Add passenger to the flight
      passenger.flights.add(flight)

      # Redirect user to flight page
      return HttpResponseRedirect(reverse("flight", args=(flight.id,)))


def flight(request, flight_id):

   flight = Flight.objects.get(id=flight_id)
   passengers = flight.passengers.all()
   non_passengers = Passenger.objects.exclude(flights=flight).all()
   return render(request, "flights/flight.html", {
      "flight": flight,
      "passengers": passengers,
      "non_passengers": non_passengers
})

Now, in views.py, we’ll add three functions:

def index(request):
   # If no user is signed in, return to login page:
   if not request.user.is_authenticated:
      return HttpResponseRedirect(reverse("login"))
   return render(request, "users/user.html")

def login_view(request):
   if request.method == "POST":
      # Accessing username and password from form data
      username = request.POST["username"]
      password = request.POST["password"]

      # Check if username and password are correct, returning User object if so
      user = authenticate(request, username=username, password=password)

      # If user object is returned, log in and route to index page:
      if user:
         login(request, user)
         return HttpResponseRedirect(reverse("index"))
      # Otherwise, return login page again with new context
      else:
         return render(request, "users/login.html", {
            "message": "Invalid Credentials"
      })
   return render(request, "users/login.html")

def logout_view(request):
   logout(request)
   return render(request, "users/login.html", {
      "message": "Logged Out"
   })

On consciousness

Consciousness is our newest stage, the latest feature of what is organic; it is, therefore, what we have of less finished and robust. It surprises me that we treat consciousness as a unit of the organism, isolating it from every other biological asset with a ridiculous overestimation in such a way that prevents us from developing it even further, as we believe we already have it.

On passion

There is an interesting pattern of psyche that can utterly be achieved by carrying an inexcusably-wide open heart, that'll minimise the illusion of control to near-zero. Passion is a self-indulging, highly inflammable fuel. It allows a special peace of mind which can only materialize from the sweet taste of purpose. It is right there - where the limitations of logic no longer allow us to acknowledge because of its extreme subtlety - that uninhabited instincts as such allow us to begin the realm of fullness; and the sensation of having now entered this realm arouses the excitement of all other instincts, previously limited, optimizing mental energy in a self-sufficient process.

On truth

The vast majority of us are always bursting with absolute truths and abhor doubts - like a disease - finding, sooner or later, a moment of emergency to which we are moved to heal them. We seek, in fact, a desperate antidote: certainty, which generates a pleasant sensation, whereas doubt does not. Doubt is an anxious idea that causes discomfort. Now, beings do not live in discomfort, they survive. So there is science; the goal of science would be to provide the pleasure of the methodology, certainties, and identified patterns. No, this is hardly binary: doubt and certainty are so entangled in the process of discovery that those who want to taste the most of one must taste the most of the other.

10 05 21
On space exploration

Depending on the observable time window, the conception of life might turn out as an improbable or an inevitable phenomenon (i.e. a carbon molecule making crude copies of itself). Intelligent life, however, requires a set of genetic malleability and random environmental episodes in a very specific arrangement to make possible the development of a cerebral cortex - which few species, on a planet of millions of them, have managed to achieve. When one contemplates the improbability of our existence (that is, matter and energy developing consciousness) plus the microscopic historical slice which our civilization is embedded within (the "habitable interval") and all the potential sources of extinction (including coming from ourselves), a feeling of urgency to maintain the lineage of our species suddenly arises. By deeply comprehending this idea, the focus of humanity might hopefully converge towards the preservation of the planet and space exploration. That could create a planetary human bonding - the first gift presented by the cosmos to the cosmos.

11 07 21
On superdeterminism

Does a nucleic acid capsule have consciousness?
A rabies virus activate a dog's rage by infecting its limbic system while turning off the instinct to swallow saliva, enhancing transmission by biting. This is a fairly complex set of skills but not necessarily consciousness: rather a behaviour pattern encoded through millions of years of natural selection.

Does a bee have free will?
Bees, compared to viruses, are much more developed beings. They can fly, eat, excrete, smell, see, mate. When a bee dies, it releases oleic acid which alarms other bees to take the dead one out of the hive. This is useful from an evolutionary point of view. However, perfectly healthy bees that mistakenly have come in contact with the acid are also expelled from the beehive, against their will. This is not useful from an evolutionary point of view but neither is it disastrous enough to change their coded behavioural lineage by natural selection. Bee perceptions are adequate to bee means.

What does this say about the human species? Is there anything that makes us distinct or special enough to make our thoughts really "ours" - and not codes resulted from a causally correlated evolutionary loophole?
Is there really anything that's our alone? Chimpanzees do trading. Ants have social structures. Elephants care for their offspring until adolescence. Octopuses use tools. A gazelle performs an act of heroism by drawing the attention of a jaguar and give its close kins time to flee. Macaque monkeys practice empathic gestures of kindness even with nonrelatives. Bowerbirds make art and decorate their nests. Wasps enslave their prey. Fireflies betray their comrades by hacking their light pattern to turn off the females and increase their own chances. Plenty of animals even kiss each other.

Plato defined man as "featherless bipeds". Perhaps we are also the result of pure superdeterminism and DNA dictate our destiny. Perhaps the only characteristic that distinguishes humans from other animals is this neurotic need to feel special.
